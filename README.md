# Open Hearts - Волонтёрская платформа

Платформа для связи волонтёров и организаторов событий, которая упрощает процесс поиска и записи на волонтёрские события, а также процесс поиска волонтёров.

## 🎯 Возможности

### Для волонтёров:
- 📋 Просмотр доступных волонтёрских событий
- 🔍 Поиск событий по навыкам и городу
- ✍️ Запись на интересующие события
- 👤 Создание профиля с указанием навыков
- 📅 Просмотр своих записей на события

### Для организаторов:
- ➕ Создание волонтёрских событий
- 📝 Управление заявками волонтёров (одобрение/отклонение)
- 🔎 Поиск волонтёров по навыкам и городу
- 👥 Просмотр профилей волонтёров

## 🚀 Технологии

- **Backend**: Django 5.2.8
- **Frontend**: HTML, Tailwind CSS (CDN)
- **База данных**: SQLite3
- **Языки**: Python 3.8+, Django Templates

## 📦 Установка

### Требования

- Python 3.8+
- pip

### Шаги установки

1. **Клонируйте репозиторий и перейдите в директорию проекта**
   ```bash
   cd volunteer-platform-main
   ```

2. **Создайте виртуальное окружение**
   ```bash
   python -m venv venv
   ```

3. **Активируйте виртуальное окружение**
   
   Windows (PowerShell):
   ```powershell
   .\venv\Scripts\Activate.ps1
   ```
   
   Windows (CMD):
   ```cmd
   venv\Scripts\activate
   ```
   
   Linux/Mac:
   ```bash
   source venv/bin/activate
   ```

4. **Установите зависимости**
   ```bash
   python -m pip install -r requirements.txt
   ```

5. **Примените миграции**
   ```bash
   python manage.py migrate
   ```

6. **Создайте начальные данные (навыки)**
   ```bash
   python manage.py setup_initial_data
   ```

7. **Создайте суперпользователя (опционально)**
   ```bash
   python manage.py createsuperuser
   ```

8. **Запустите сервер разработки**
   ```bash
   python manage.py runserver
   ```

   Если `python manage.py ...` запускается не из `venv` (например, ошибка `No module named 'django'`), используйте интерпретатор окружения напрямую:
   ```powershell
   .\venv\Scripts\python.exe manage.py runserver
   ```

9. **Откройте в браузере**
   ```
   http://127.0.0.1:8000/
   ```

## 👤 Админ-доступ

Суперпользователь создаётся вручную командой:
```bash
python manage.py createsuperuser
```

После создания вход в админ-панель: http://127.0.0.1:8000/admin/

## 🌐 Основные страницы

| URL | Описание |
|-----|----------|
| `/` | Главная страница - список событий с фильтрами |
| `/register/` | Регистрация (выбор роли: волонтёр/организатор) |
| `/login/` | Вход в систему |
| `/profile/` | Профиль пользователя (редактирование) |
| `/events/<id>/` | Детали события |
| `/events/create/` | Создание события (только для организаторов) |
| `/events/<id>/register/` | Запись на событие (только для волонтёров) |
| `/events/<id>/manage/` | Управление заявками (только для организатора события) |
| `/volunteers/` | Поиск волонтёров (только для организаторов) |
| `/my-events/` | Мои события |
| `/admin/` | Админ-панель Django |

## 🎨 Модели данных

### Skill (Навык)
- Навыки волонтёров (экология, медицина, IT и т.д.)

### UserProfile (Профиль пользователя)
- Роль: волонтёр или организатор
- Навыки, контакты, биография
- Автоматически создаётся при регистрации

### Event (Событие)
- Информация о событии
- Организатор
- Требуемые навыки
- Лимит участников

### EventRegistration (Заявка на событие)
- Заявки волонтёров на события
- Статусы: pending, approved, rejected, cancelled

## 🎯 Использование

### Регистрация

1. Перейдите на `/register/`
2. Выберите роль (Волонтёр или Организатор)
3. Заполните форму регистрации
4. После регистрации автоматически создаётся профиль

### Создание события (для организаторов)

1. Войдите как организатор
2. Перейдите на `/events/create/`
3. Заполните информацию о событии
4. Выберите требуемые навыки
5. Установите лимит участников

### Поиск волонтёров (для организаторов)

1. Перейдите на `/volunteers/`
2. Используйте фильтры (навыки, город)
3. Просматривайте профили волонтёров
4. Связывайтесь с ними для участия в событиях

## 📝 Настройки

Основные настройки находятся в `volunteer/settings.py`:

- `LANGUAGE_CODE = 'ru-ru'` - Русский язык
- `TIME_ZONE = 'UTC'` - Часовой пояс
- `LOGIN_URL = 'login'` - URL для входа
- `LOGIN_REDIRECT_URL = 'event_list'` - Перенаправление после входа

## 🐛 Решение проблем

### Ошибка миграций
Если возникают проблемы с миграциями:
```bash
python manage.py makemigrations
python manage.py migrate
```

### Ошибка "no such table"
Удалите `db.sqlite3` и выполните миграции заново:
```bash
rm db.sqlite3  # Linux/Mac
del db.sqlite3  # Windows
python manage.py migrate
python manage.py setup_initial_data
```

### Проблемы с виртуальным окружением
Убедитесь, что виртуальное окружение активировано перед установкой зависимостей и запуском команд.
